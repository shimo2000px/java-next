現在のPRのコードをAIレビューし、結果をGitHub PRコメントとして投稿します。

## 手順

1. 対象PRを特定する:
   - `$ARGUMENTS` にPR番号があればそれを使用
   - なければ `gh pr view --json number` で現在のブランチのPRを取得

2. 差分を取得する:
   ```
   gh pr diff <PR番号>
   ```

3. 以下の観点でコードをレビューする:
   - **正確性**: バグ、ロジックエラー、エッジケース漏れ
   - **セキュリティ**: SQL injection、XSS、認証・認可の欠陥、秘密情報の露出
   - **パフォーマンス**: N+1クエリ、不要な処理、メモリリーク
   - **可読性**: 命名、複雑度、コメントの適切さ
   - **テスト**: テスト漏れ、テストの品質

4. レビュー結果を以下のフォーマットでまとめ、`gh pr comment <PR番号> --body "..."` で投稿する:

```
## 🤖 AIコードレビュー

### 総評
<!-- 全体的な評価 (良い/要修正/ブロッカーあり) -->

### 🔴 必須修正 (ブロッカー)
<!-- セキュリティ問題・重大バグなど。なければ「なし」 -->

### 🟡 改善推奨
<!-- パフォーマンス・可読性など。なければ「なし」 -->

### 🟢 良い点
<!-- 特に優れているコードや設計 -->

### 📝 その他コメント
<!-- 軽微な提案など -->

---
*Reviewed by Claude Code*
```

5. 投稿したコメントのURLを表示する。

GitHubにIssueを作成します。

## 手順

1. `$ARGUMENTS` からIssueの概要を把握する。引数がない場合はユーザーに確認する。
2. Issueの種類を判断する:
   - バグ報告 → bug_report テンプレート
   - 機能要望 → feature_request テンプレート
3. 以下のテンプレートをベースに内容を埋めて `gh issue create` で作成する。

### バグ報告テンプレート
```
## バグの概要
<!-- 何が起きているか簡潔に -->

## 再現手順
1. 
2. 
3. 

## 期待する動作
<!-- 本来どうなるべきか -->

## 実際の動作
<!-- 実際に何が起きているか -->

## 環境
- OS: 
- バージョン: 

## 追加情報
<!-- スクリーンショット、ログなど -->
```

### 機能要望テンプレート
```
## 概要
<!-- 追加したい機能を簡潔に -->

## 背景・目的
<!-- なぜこの機能が必要か -->

## 詳細仕様
<!-- 具体的な動作・UI・APIなど -->

## 受け入れ条件
- [ ] 
- [ ] 

## 追加情報
<!-- 参考リンク、モックアップなど -->
```

4. 作成後、IssueのURLを表示する。
